# 简介

设计模式，即Design Patterns，是指在软件设计中，被反复使用的一种代码设计经验。使用设计模式的目的是为了可重用代码，提高代码的可扩展性和可维护性。

设计模式这个术语是上个世纪90年代由`Erich Gamma`、`Richard Helm`、`Raplh Johnson`和`Jonhn Vlissides`四个人总结提炼出来的，并且写了一本《Design Patterns》的书。这四人也被称为四人帮（GoF）。
